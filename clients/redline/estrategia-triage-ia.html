CRMinbox no tiene conector nativo en Make.com. La solución: webhook de CRMinbox → Make.com + módulos HTTP para llamar la API. Funciona igual de bien.

Workaround estándar

El sistema

Flujo de triage inteligente

Desde que el cliente envía un mensaje hasta que el ticket llega al agente correcto — completamente automatizado.

0

Mensaje recibido en CRMinbox

Llega por WhatsApp, email o web chat. CRMinbox dispara webhook a Make.com con el payload del mensaje.

1

Resolución de identidad

Make.com busca el teléfono en CRMinbox. Si hay match → extrae nombre, plan, tipo de servicio, historial de tickets, estado de cuenta. Si no hay match → pide número de cuenta o dirección en un solo mensaje.

2

Saludo contextual automático

El bot envía un primer mensaje personalizado antes de que toque un agente humano.

"Hola Carlos, soy el asistente de RedLine. Vi que tienes el plan 50MB en Laureles. ¿En qué te puedo ayudar hoy?"
3

Clasificación por IA (Claude API)

Make.com envía el mensaje + contexto del cliente a Claude API. El modelo devuelve JSON estructurado con categoría, subcategoría, prioridad, sentimiento, riesgo de churn y si aplica autoservicio.

{ "category": "technical_t1", "subcategory": "slow_speed", "confidence": 0.94, "priority": "medium", "sentiment": -0.5, "churn_risk": "medium", "pqrs_flag": false, "self_service_applicable": false, "agent_briefing": "Cliente lleva 8 meses, 2 tickets técnicos este mes" }
4

Verificación de completitud

Si confianza ≥ 85% → procede. Si está entre 60–84% → hace una pregunta aclaratoria al cliente. Si <60% → ruta a cola general con flag "necesita clasificación manual". Máximo 2 preguntas de clarificación.

5

Creación y enriquecimiento del ticket

Make.com actualiza el ticket en CRMinbox: pone el tag de categoría, asigna prioridad, agrega resumen generado por IA al ticket y calcula el timestamp límite del SLA.

6

Ruteo al agente correcto

Asigna al agente menos cargado del equipo correcto (Soporte T1 / Facturación / Ventas / Retención / Calidad). Notifica al agente con briefing del cliente. Confirma al cliente con número de caso y tiempo esperado de respuesta.

"Tu caso fue asignado a nuestro equipo de Soporte Técnico. Un agente te contactará en máximo 30 minutos. Caso #4821 — Guardado ✓"
7

Escalaciones automáticas

Palabras como "SIC", "demanda", "Superintendencia" → tag PQRS inmediato + alerta Calidad. Sentimiento muy negativo → tag "Riesgo churn alto". Mismo problema 3+ veces en 30 días → escala a T2. SLA al 80% → alerta supervisor.

Más allá del triage

8 automatizaciones adicionales

El triage es el core, pero hay 8 flujos más que pueden construirse sobre la misma infraestructura.

📡

Alertas proactivas de averías

Cuando el NOC detecta una caída, el sistema identifica todos los clientes afectados en la zona y les envía un WhatsApp explicando la situación y el tiempo estimado de solución. Suprime la creación de tickets duplicados durante el evento.

Deflecta 60–80% tickets en averías
📅

Ciclo de vida de visitas técnicas

T-24h: recordatorio + confirmación (SÍ/NO). Si NO: reagendar. T-1h: "tu técnico está en camino". Post-visita: cierre automático del ticket. T+2h: encuesta de satisfacción del técnico.

Reduce no-shows
💳

Recordatorios de pago automatizados

D-5 y D-1 antes del vencimiento: WhatsApp con monto y link de pago. D+1 y D+3: aviso de mora. D+7: advertencia de suspensión. Pago confirmado: mensaje de agradecimiento automático.

Reduce cartera morosa
⭐

Encuesta CSAT post-resolución

2h después de cerrar un ticket: "¿Quedó resuelto? 1=Sí / 2=Parcialmente / 3=No". Si responde 3 → reabre ticket + alerta supervisor. Si responde 2 → flag calidad.

Mide satisfacción real
🏠

Pipeline de nuevas instalaciones

Lead → verificación de cobertura → envío de planes → agendamiento de instalación → confirmaciones → guía de bienvenida post-instalación → check T+48h.

Experiencia completa automatizada
🚨

Scoring de riesgo de churn

Cada semana el sistema calcula el riesgo de cada cliente: tickets recientes, sentimiento histórico, pagos tardíos, antigüedad. Riesgo Alto/Crítico → alerta a Retención para contacto proactivo antes de que el cliente pida cancelar.

Retención preventiva
🔁

Detección de problemas recurrentes

3+ tickets del mismo tipo en 30 días para la misma cuenta → auto-escala a T2 para análisis de causa raíz + flag en dashboard para Calidad. Evita que el cliente llame infinitas veces por lo mismo.

Mejora FCR
🤖

Autoservicio inteligente

Antes de asignar agente, la IA detecta si la solicitud tiene solución de autoservicio: guía de reinicio de router, link de descarga de factura, link de pago, consulta de cobertura. Solo si el cliente confirma que no le funcionó → crea ticket.

Deflecta 15–25% del volumen
Arquitectura

Stack tecnológico

Cada pieza tiene un rol específico. La arquitectura es modular — si CRMinbox tiene limitaciones de API, hay plan B listo.

CRMinbox Sistema de registro + interfaz de agentes

Todos los tickets viven aquí. Los agentes trabajan desde la interfaz de CRMinbox. La IA enriquece los tickets pero no reemplaza el CRM — lo hace más inteligente.

  • Requiere: webhook en nuevas conversaciones + API para actualizar tickets
  • Acción: confirmar disponibilidad de API y webhooks con soporte CRMinbox

Make.com Capa de orquestación (sin código)

Conecta todos los sistemas y corre los 9 escenarios de automatización. Visual, mantenible, no requiere servidor ni equipo de desarrollo para ajustes.

  • Costo: $29–$99 USD/mes según volumen de operaciones
  • Escenarios: Triage, SLA, Alertas averías, Recordatorios visitas, CSAT, Pagos, Pipeline ventas, Churn scoring, Autoservicio

Claude API (Anthropic) Motor de clasificación

No es un chatbot conversacional — es la capa de inteligencia que clasifica cada mensaje y devuelve JSON estructurado. Rápido, preciso, 100% controlable.

  • Modelo: Claude Haiku (triage rápido y económico) / Claude Sonnet (PQRS y casos complejos)
  • Costo estimado: $30–$120 USD/mes según volumen de mensajes
  • Output: JSON con categoría, prioridad, sentimiento, riesgo churn, briefing de agente

WhatsApp Business API Canal de comunicación con clientes

Todos los mensajes salientes automatizados van por aquí: confirmaciones, recordatorios, encuestas, alertas de avería, avisos de pago.

  • Proveedor recomendado: 360dialog (mejor integración con Make.com, amigable con Colombia)
  • Requiere cuenta de WhatsApp Business API verificada por Meta

Google Looker Studio / Metabase Dashboard de métricas

Conectado a los logs de Make.com (Google Sheets o Supabase). Muestra KPIs en tiempo real: colas, SLA, CSAT, volumen por categoría, performance de agentes.

  • Looker Studio: gratuito
  • Metabase: gratuito self-hosted

Plan B — Si CRMinbox no tiene API robusta: Se agrega Airtable o Supabase como base de datos intermedia. Make.com escribe todos los tickets ahí. CRMinbox sigue siendo la UI para los agentes, pero el motor de automatización opera independiente. Sin bloqueo.

Implementación

3 fases de desarrollo

De cero a sistema predictivo completo en 6 meses. Cada fase entrega valor inmediato antes de la siguiente.

Fase 1 — Quick Wins

Triage básico y ruteo funcionando en semanas, no meses

Semanas 1–4
  • Auditoría de API de CRMinbox: test de webhook, actualización de tickets, lectura de contactos
  • Conectar número de WhatsApp Business API a 360dialog
  • Crear equipos y colas en CRMinbox (Soporte / Facturación / Ventas / Calidad)
  • Make.com Escenario A: nuevo mensaje → acuse → ticket → asignación a cola general
  • Integrar Claude API para clasificación con prompt de taxonomía ISP
  • Auto-ruteo por categoría a cola correcta + tag en ticket
  • Detección de keywords PQRS y alerta inmediata a Calidad
  • Dashboard básico en Google Looker Studio con volumen y FRT
  • Capacitación a agentes en nuevo flujo de tickets pre-clasificados

Resultado: 80%+ de tickets auto-clasificados y pre-ruteados. Tiempo de respuesta inicial baja de 45–90 min a <5 min (acuse inmediato). Agentes dejan de hacer triage manual.

Fase 2 — Automatización Completa

Cada punto de contacto repetitivo automatizado

Meses 1–3
  • Ciclo completo de visitas técnicas: recordatorio T-24h, T-1h, encuesta post-visita
  • CSAT automático en resolución de tickets + reapertura si cliente dice "no resuelto"
  • Recordatorios de pago automatizados (D-5, D-1, D+1, D+3, D+7)
  • Autoservicio: guías de router, links de factura/pago antes de crear ticket
  • Sistema de alertas de avería: NOC activa → broadcast por zona + supresión de tickets duplicados
  • Pipeline de nueva instalación: lead → cobertura → planes → agendamiento → bienvenida
  • Churn scoring semanal + alerta proactiva a Retención para casos Alto/Crítico
  • Detección de problemas recurrentes (3+ tickets mismo tipo en 30 días) → auto-escala T2
  • Dashboard v2: CSAT, churn rate, retention, deflection, SLA por equipo, ranking agentes

Resultado: 25–35% reducción de volumen de tickets. CSAT mejora 0.5–1.0 puntos. Tasa de churn baja 0.5–1.0%. Cada agente libera 2–3h/día de trabajo manual.

Fase 3 — IA Predictiva

De reactivo a preventivo: detectar problemas antes de que el cliente llame

Meses 3–6
  • Integración con herramienta de monitoreo de red (PRTG / Zabbix / OLT): alertas proactivas antes de que el cliente note la falla
  • Resolución full-AI Tier 0: router restart, descarga de factura, booking de citas — sin agente humano
  • Briefing inteligente por cliente antes de cada conversación: historial, plan, riesgo, recomendación de oferta
  • Análisis de sentimiento histórico mensual por zona, producto y agente
  • Canal de voz: integración con AI de voz (Bland.ai / Vapi) para llamadas salientes automatizadas
  • Campañas proactivas segmentadas: clientes en plan antiguo → oferta de upgrade personalizada
  • Informe ejecutivo mensual con ROI calculado: costo por ticket, ahorros por automatización, revenue protegido

Resultado: Tier 0 (IA resuelve sola) maneja 30–40% del volumen total. Tickets inbound bajan 15–20% adicional por prevención proactiva. ROI completamente documentado y justificable.
